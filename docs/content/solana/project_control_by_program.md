# Solana/在主网发行您的代币/由程序控制的代币

要实现一个空投合约, 我们首先得满足这个空投合约本身持有这些代币. 空投合约应当像普通钱包账户一样, 可以接收代币转账, 也可以转账代币给其它用户.

在 solana 中, 代币的转账并不是直接发生在用户的主账户与接收账户之间, 而是通过关联代币账户来完成. 每个用户的关联代币账户是与其主账户相联系的, 而程序也可以拥有关联代币账户来执行代币的转账与管理.

通常来讲, 要实现由程序控制的代币这个功能, 需要有以下三个账户层级.

* 程序账户. 这是一个智能合约账户, 它控制着代币的空投. 程序账户通过 `invoke` 或 `invoke_signed` 来与其他账户进行交互.
* 程序 pda 账户. 程序账户通常不允许直接拥有代币, 它必须通过派生一个 pda 账户来存储代币. 这些 pda 账户的地址是通过程序地址和特定种子派生出来的.
* 关联代币账户. 关联代币账户由程序的 pda 账户派生而来.

简单来说, 账户之间的关系: 程序账户 -> 程序 pda 账户 -> 关联代币账户.

真正拥有代币所有权的是程序 pda 账户, 但得以于 pda 账户的特性, 程序账户可以代替其 pda 账户签名, 因此程序账户也就变相拥有了代币的所有权.
